## DTO Shapes

### CreateDocumentDto (multipart body — non-file fields)
```typescript
{
  title: string;             // @IsString @MaxLength(200) @MinLength(1)
  category: DocumentCategory; // @IsEnum(DocumentCategory)
  description?: string;       // @IsOptional @IsString @MaxLength(2000)
  isPublic?: boolean;         // @IsOptional @IsBoolean — default true if omitted
                              // class-transformer @Transform(({ value }) => value === 'true' || value === true)
}
// The `file` field is bound via @UploadedFile() in the controller — not part of this DTO class.
```

### UpdateDocumentDto (multipart body — non-file fields, all optional)
```typescript
{
  title?: string;             // @IsOptional @IsString @MaxLength(200) @MinLength(1)
  description?: string;       // @IsOptional @IsString @MaxLength(2000)
  category?: DocumentCategory; // @IsOptional @IsEnum(DocumentCategory)
  isPublic?: boolean;         // @IsOptional @IsBoolean (with the same Transform as CreateDocumentDto)
}
// `file` is optional on PATCH; presence triggers version bump + new file write + event.
// `id`, `uploadedById`, `version`, `sourceUrl`, `sizeBytes`, `mimeType`, `createdAt`, `updatedAt`
// are immutable from the client perspective and are silently ignored by the ValidationPipe whitelist.
```

### ListDocumentsQueryDto
```typescript
{
  page?: number;              // @IsOptional @Type(() => Number) @IsInt @Min(1), default: 1
```

## DocumentStorage Interface

The storage backend abstraction lives at `apps/social/src/modules/documents/storage/document-storage.interface.ts`:

```typescript
export const DOCUMENT_STORAGE = Symbol('DOCUMENT_STORAGE');

export interface DocumentStorage {
  /**
   * Persist a file buffer at the given key. Overwrites if the key already exists.
   * Throws StorageWriteError on failure — caller MUST surface 503 StorageUnavailable.
   */
  put(key: string, data: Buffer, mimeType: string): Promise<void>;

  /**
   * Open a readable byte stream for the given key. Throws StorageKeyNotFound
   * if the key does not exist — caller MUST surface 410 DocumentFileMissing.
   */
  get(key: string): Promise<NodeJS.ReadableStream>;

  /**
   * Best-effort delete. Resolves silently if the key did not exist.
   * Throws StorageWriteError on permission/IO errors — caller logs but does NOT roll back.
   */
  delete(key: string): Promise<void>;

  /**
   * Cheap existence check. Used by the download endpoint to decide 410 vs streaming.
   */
  exists(key: string): Promise<boolean>;
}
```

The default `FilesystemDocumentStorage` implementation (see plan T-03) resolves keys under `DOCUMENT_STORAGE_PATH` and enforces a `path.resolve()` allowlist check to prevent any escape.

## State Transitions

```
POST /documents (multipart file + dto)
  → validate MIME + size + non-empty
  → write file to storage (key = documents/{newId}/v1/{sanitizedName})
  → INSERT row with version = 1, sourceUrl = key
  → emit document.uploaded
  → 201 DocumentResponse

PATCH /documents/:id { metadata only, no file }
  → UPDATE row (version unchanged)
  → 200 DocumentResponse, no event

PATCH /documents/:id (multipart file + optional metadata)
  → validate new file
  → write new file to storage (key = documents/{id}/v{N+1}/{sanitizedName})
  → UPDATE row with version = N+1, sourceUrl = new key, mimeType/sizeBytes refreshed, optional metadata applied
  → best-effort delete OLD file (failure logged, NOT a rollback)
  → emit document.uploaded with version = N+1
  → 200 DocumentResponse

DELETE /documents/:id
  → DELETE row (hard delete)
  → best-effort delete file from storage
  → emit document.deleted
  → 204 No Content
```

## Visibility Filtering Logic

Applied server-side in `DocumentsService.findAll()`:

```
caller_roles = jwt.roles
is_admin = 'HR_ADMIN' ∈ caller_roles

WHERE
  (is_admin OR isPublic = TRUE)            -- visibility
  AND (?category IS NULL OR category = ?category)
  AND (?search IS NULL OR title ILIKE '%' || ?search || '%')

ORDER BY createdAt DESC, id ASC
LIMIT ?pageSize OFFSET ((?page - 1) * ?pageSize)
```

For `findOne` and `download`:
```
SELECT row WHERE id = ?id
IF row IS NULL → 404 DocumentNotFound
IF row.isPublic = FALSE AND NOT is_admin AND NOT is_system → 404 DocumentNotFound
(For download only:)
IF NOT storage.exists(row.sourceUrl) → 410 DocumentFileMissing
ELSE → stream
```

`is_system = 'SYSTEM' ∈ caller_roles` — only checked on the download endpoint. Other endpoints reject SYSTEM JWTs by virtue of their `@Roles(...)` lists not including 'SYSTEM'.

## Error Code Catalog

All error codes returned by the documents module:

| Code | Status | Where |
|------|--------|-------|
| `UnsupportedMimeType` | 400 | POST/PATCH — MIME not in whitelist |
| `FileTooLarge` | 400 | POST/PATCH — file.size > DOCUMENT_MAX_SIZE_BYTES |
| `EmptyFile` | 400 | POST/PATCH — file.size === 0 |
| `MissingFile` | 400 | POST — `file` field absent from multipart body |
| `DocumentNotFound` | 404 | GET/PATCH/DELETE — id does not exist OR row hidden by visibility |
| `DocumentFileMissing` | 410 | GET /:id/download — row exists but storage key missing |
| `StorageUnavailable` | 503 | POST/PATCH — `DocumentStorage.put` threw |

These exact strings are returned in the response body's `message` field so the frontend's `onError` mapping (FR-031) can render human-readable messages.
